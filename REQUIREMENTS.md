## 1. Scope Overview

### Goals

- Convert today's hardcoded `src/lib/data.ts` portfolio into a fully admin-editable CMS.
- Every piece of visible chrome — name, title, tagline, navbar, footer, social icons, logo, favicon, OG image, theme colors, section ordering — admin-editable.
- Public landing page server-rendered for SEO + fast TTFB.
- Admin panel with React Query, skeletons, toasts.
- Centralized theming (single-file palette change via DB or `globals.css`).
- MongoDB for all data, GridFS for media.
- Three.js 3D scenes stay code-driven (only their **enabled / disabled** state is dynamic).

### Scope boundary

**In scope:**
- 1 landing page (10 toggleable sections, the existing visual design)
- 17 admin CMS modules (see §3)
- Public REST API for landing-page data
- Admin REST API with session auth
- Responsive (mobile / tablet / desktop) — unchanged from current
- SEO (meta, sitemap, schema.org Person, dynamic OG, dynamic favicon)
- MongoDB Atlas (or self-hosted) deployment
- Docker-Compose local MongoDB
- Universal `<Loader />` and admin skeletons

**Out of scope (v1):**
- Multilingual content
- Blog / article system
- Newsletter / email collection
- Payment / commerce
- Multi-tenant (single owner only)

## 2. Public Surface Specification

The public surface is a **single landing page** at `/` composed of 10 sections. The landing page is a server component. Each section is fetched in parallel via `lib/queries/*` helpers, then rendered. Sections can be hidden + reordered via `home_sections`.

### 2.1 Site chrome (every page)

| Element | Source collection / field |
|---|---|
| `<title>` template | `site_settings.site_title` + `site_settings.site_title_template` |
| `<meta description>` | `site_settings.site_description` |
| Favicon | Generated by `app/icon.tsx` from `site_settings.favicon_glyph` + `favicon_bg_gradient` (or `favicon_image_id`) |
| OG image | Generated by `app/opengraph-image.tsx` from `site_settings.og_*` fields |
| Theme color (browser chrome) | `site_settings.theme_color_dark` / `theme_color_light` |
| JSON-LD `<Person>` schema | `site_settings.json_ld_*` |
| Navbar items | `nav_menu_items` filter `location: "header"` |
| Footer copyright + links | `nav_menu_items` filter `location: "footer"` + `site_settings.footer_copyright_template` |
| Social icons | `site_settings.socials[]` |
| Theme tokens | `site_settings.theme_dark` + `theme_light` objects injected into `<style>` in `layout.tsx` |
| Smooth-scroll enabled flag | `site_settings.enable_smooth_scroll` |
| Custom cursor enabled | `site_settings.enable_custom_cursor` |

### 2.2 Landing page sections (`/`)

Each is a server component reading from a model. Order + visibility from `home_sections`.

| # | Section key | Component | Data source |
|---|---|---|---|
| 1 | `hero` | `HeroSection` | `hero_content` (singleton) — name, title, tagline, primary CTA, secondary CTA, `enable_3d_canvas` flag |
| 2 | `about` | `AboutSection` | `about_content` (singleton) — summary, `profile_image_id`, stats array (`{ label, value, suffix }`) |
| 3 | `experience` | `ExperienceSection` | `experiences[]` — sorted by `orderIndex` |
| 4 | `skills` | `SkillsSection` | `skills[]` grouped by `category`, plus `skill_categories[]` for tab order |
| 5 | `projects` | `ProjectsSection` | `projects[]` — sorted by `orderIndex` |
| 6 | `publications` | `PublicationsSection` | `publications[]` — sorted by `year desc` then `orderIndex` |
| 7 | `open-source` | `OpenSourceSection` | `open_source_contributions[]` with embedded `contributions[]` |
| 8 | `awards` | `AwardsSection` | `awards[]` + `certifications[]` |
| 9 | `education` | `EducationSection` | `education` (singleton) + `community_involvement[]` |
| 10 | `contact` | `ContactSection` | `site_settings.contact_*` + `site_settings.enable_3d_contact` |

### 2.3 Behavior changes vs. current

- **Server-component sections.** Today, every `*Section.tsx` is `"use client"`. Migration extracts data fetching into a server-component shell (`HeroSection` becomes `HeroSection.server.tsx`) and pushes Framer Motion/Lenis interactivity into a small `"use client"` child (`HeroSection.client.tsx`).
- **Navbar / Footer load nav items from server.** A new `<NavbarShell>` server component fetches `nav_menu_items` and passes them as props to the existing `<Navbar>` client component (which keeps the magnetic-link animations).
- **Universal loader.** A single `<UniversalLoader />` is used by every `loading.tsx` on the public side. It animates the brand wordmark from `site_settings.brand_short` so even loaders look on-brand.
- **Smooth-scroll + custom cursor** become toggleable via `site_settings`. Default ON.

## 3. Admin Modules Specification

All admin pages live under `app/admin/<route>/`. Every page is `"use client"` and consumes React Query. Every page has its own `loading.tsx` skeleton and `error.tsx`.

The sidebar groups modules into **Site**, **Content**, **Ops**, **Admin**.

### Site (chrome / global)

1. **Site Settings** (`/admin/settings`)
   - Tabs: Branding · SEO · Contact · Social · Theme · OG/Favicon · Toggles
   - Branding: site_title, site_title_template, brand_short, brand_full, logo upload (optional), tagline override
   - SEO: site_description, keywords, site_url, twitter_handle
   - Contact: email, phone, location
   - Social: array editor — platform, url, label, icon name (from `lib/icons.ts`)
   - Theme: color picker × 12 vars × dark/light; font_sans, font_display, font_mono (dropdown of allowed `next/font/google` faces)
   - OG/Favicon: og_title, og_subtitle, og_chips (string array), og_bg_gradient, favicon_glyph, favicon_bg_gradient, OR upload `og_image` / `favicon_image`
   - Toggles: enable_3d_hero, enable_3d_contact, enable_smooth_scroll, enable_custom_cursor, enable_scroll_progress, dark_mode_default
   - JSON-LD: name, jobTitle, url, sameAs[], knowsAbout[], alumniOf, address fields
2. **Navigation** (`/admin/navigation`)
   - Tabs: Header · Footer
   - Per tab: drag-to-reorder list of `nav_menu_items`, CRUD label/href/icon/isActive/opensInNewTab
3. **Section Order** (`/admin/sections`)
   - Drag-to-reorder list of `home_sections` (10 keys, fixed); toggle `isVisible`

### Content (per section)

4. **Hero** (`/admin/hero`) — singleton form: name, role title, tagline, primary CTA (label + href), secondary CTA, animation delays.
5. **About** (`/admin/about`) — singleton: summary paragraph, profile image upload, stats[] CRUD (label/value/suffix).
6. **Experience** (`/admin/experience`) — list + create/edit modal: company, role, location, period, type (full-time | remote | freelance), bullets[], tech[], orderIndex.
7. **Skills** (`/admin/skills`) — split view: left = `skill_categories[]` (drag-reorder); right = `skills[]` filtered by selected category, CRUD name + production flag.
8. **Projects** (`/admin/projects`) — list table + edit page (slug, title, subtitle, description, tech[], metric, link, github, orderIndex, cover image optional).
9. **Publications** (`/admin/publications`) — list table + edit: title, authors, venue, year, doi, description, orderIndex.
10. **Open Source** (`/admin/open-source`) — parent record (project, organization, description, repoUrl) with nested `contributions[]` CRUD (id, title, severity, description, url).
11. **Awards** (`/admin/awards`) — list table: title, event, rank, orderIndex.
12. **Certifications** (`/admin/certifications`) — list table: title, issuer, link, orderIndex.
13. **Education** (`/admin/education`) — singleton + `community_involvement[]` CRUD.

### Ops

14. **Media Library** (`/admin/media`) — grid view of every GridFS upload, filter by mimeType, copy URL, delete (with reference check warning).
15. **Inquiries** (`/admin/inquiries`) — list of contact-form submissions (mark read / archive). Public POST `/api/public/inquiries` writes here.

### Admin

16. **Users** (`/admin/users`) — super_admin only. CRUD admins, reset password, toggle isActive, role.
17. **Dashboard** (`/admin/`) — counts (sections published, projects, experiences), recent inquiries, quick links.

## 5. Database Schema

All collections in MongoDB database `myportfolio`. Every document has `_id: ObjectId`, `createdAt`, `updatedAt`. Image fields are pairs `{ url: string, mediaId: ObjectId }` (both nullable).

### Singletons (one document each)

| Collection | Notable fields |
|---|---|
| `site_settings` | one document with `{ key: "default", … all chrome fields above }` |
| `hero_content` | `name, title, tagline, primaryCta {label, href}, secondaryCta, enable3dCanvas, eyebrowText` |
| `about_content` | `summary, profileImage {url,mediaId}, profileAlt, stats[]` |
| `education` | `school, degree, grade, period, location, logoImage` |

### Collections (multi-doc)

| Collection | Notable fields |
|---|---|
| `users` | `email, name, passwordHash, role (super_admin\|admin), isActive, lastLoginAt` |
| `sessions` | `userId, token, expiresAt, userAgent, ipAddress` |
| `home_sections` | `key (unique), label, isVisible, orderIndex` — 10 fixed keys, seeded |
| `nav_menu_items` | `parentId?, label, href, icon?, location (header\|footer), orderIndex, isActive, opensInNewTab` |
| `experiences` | `company, role, location, period, type, bullets[], tech[], orderIndex, isVisible` |
| `skill_categories` | `name, slug, orderIndex` |
| `skills` | `name, categoryId, production, orderIndex` |
| `projects` | `slug (unique), title, subtitle, description, tech[], metric, link?, github?, coverImage {url,mediaId}?, orderIndex, isPublished` |
| `publications` | `title, authors, venue, year, doi?, description, orderIndex, isVisible` |
| `open_source_contributions` | `project, organization, description, repoUrl, orderIndex, contributions[{id,title,severity,description,url}]` |
| `awards` | `title, event, rank, orderIndex, isVisible` |
| `certifications` | `title, issuer, link, orderIndex, isVisible` |
| `community_involvement` | `role, org, year, description, orderIndex` |
| `inquiries` | `name, email, message, ipAddress, userAgent, status (new\|read\|archived), createdAt` |
| `media.files` / `media.chunks` | GridFS bucket `media` (file + chunk collections auto-managed by GridFSBucket) |

Indexes:
- `users.email` unique
- `sessions.token` unique
- `sessions.expiresAt` TTL index
- `projects.slug` unique
- `skills.categoryId + orderIndex`
- `nav_menu_items.location + orderIndex`
- `home_sections.key` unique
- `experiences.orderIndex`

## 6. API Layer

All responses follow `{ success: boolean; data?: T; error?: string }`.

### Auth (`/api/auth`)

| Method | Path | Auth | Body | Returns |
|---|---|---|---|---|
| POST | `/login` | none | `{ email, password }` | `{ user }` + sets cookie |
| POST | `/logout` | session | — | `{ ok: true }` clears cookie |
| GET | `/session` | session | — | `{ user }` or 401 |

### Admin (`/api/admin/*`) — all wrapped in `withAdminAuth`

For each resource: `GET /api/admin/<resource>` (list), `POST /api/admin/<resource>` (create), `GET /api/admin/<resource>/[id]` (detail), `PATCH /api/admin/<resource>/[id]` (update), `DELETE /api/admin/<resource>/[id]` (delete). Singletons skip create/list/detail/delete and only expose `GET /api/admin/<resource>` + `PATCH /api/admin/<resource>`.

| Resource | Type |
|---|---|
| `settings` | singleton |
| `hero` | singleton |
| `about` | singleton |
| `education` | singleton |
| `navigation` | collection (bulk reorder endpoint: `PATCH /api/admin/navigation/reorder`) |
| `sections` | collection (only update — `PATCH /api/admin/sections`) |
| `experience` | collection |
| `skill-categories` | collection |
| `skills` | collection |
| `projects` | collection |
| `publications` | collection |
| `open-source` | collection |
| `awards` | collection |
| `certifications` | collection |
| `community` | collection |
| `inquiries` | read-only list + PATCH status + DELETE |
| `users` | super_admin only |
| `upload` | POST multipart → GridFS; returns `{ id, url, mimeType, size }` |
| `media` | super_admin DELETE (others read via public route) |

Every admin mutation calls `revalidateTag("public-data")` (or the more granular tag) after the DB write so the public landing page reflects changes within seconds.

### Public (`/api/public/*`)

| Method | Path | Body | Notes |
|---|---|---|---|
| POST | `/inquiries` | `{ name, email, message }` | Rate-limited (5/15min by IP), Turnstile (optional) |

Public page data is **not** fetched via API — it's read directly via Mongoose in server components for SEO + perf. The public API exists only for write actions (contact form) and future client-side widgets.

### Media (`/api/media/[id]`)

| Method | Path | Notes |
|---|---|---|
| GET | `/media/[id]` | Streams GridFS file. Sets `Content-Type`, `Cache-Control: public, max-age=31536000, immutable`, ETag |

## 7. State Management

- **Server state (public):** read via Mongoose in async server components, cached with `unstable_cache(..., [key], { tags: ["<entity>"] })`. Tag invalidated via `revalidateTag()` from the admin mutation handler.
- **Server state (admin):** React Query.
- **UI state:** Zustand (`ui-store` for sidebar collapse + mobile menu; `admin-filter-store` for table filters).
- **Form state:** React Hook Form + Zod resolver.

## 8. Validation

- Every Mongoose schema validates required + length + enum.
- Every form + API uses Zod (`lib/validations/<entity>.ts`).
- Validation errors return `{ success: false, error: "<first issue message>" }`.

## 9. Theme & Design System

Default tokens stay in `app/globals.css`. Admin overrides are injected at runtime in `app/layout.tsx`:

```tsx
const settings = await getSiteSettings();
return (
  <html>
    <head>
      <style>{`:root { ${cssVarsFrom(settings.themeDark)} } .light { ${cssVarsFrom(settings.themeLight)} }`}</style>
    </head>
    …
  </html>
);
```

Editable vars (per mode):
- `--background`, `--background-secondary`, `--background-tertiary`
- `--foreground`, `--foreground-secondary`, `--foreground-tertiary`
- `--accent-blue`, `--accent-amber`, `--accent-emerald`
- `--border`

Editable fonts: `font_sans`, `font_display`, `font_mono` — values are keys into a whitelist (`Inter | Space Grotesk | JetBrains Mono | Roboto | Playfair Display | …`) defined in `lib/theme.ts`. The whitelist matters because `next/font/google` needs static imports — we can't accept arbitrary strings.

## 14. Security

- bcrypt 12 rounds.
- 7-day session, httpOnly cookie, SameSite=Lax, Secure in prod.
- Origin check on mutation methods in `withAdminAuth`.
- Rate limiter on `/api/auth/login` (5/15min) and `/api/public/inquiries` (5/15min).
- Security headers in `next.config.ts`: X-Frame-Options DENY, X-Content-Type-Options nosniff, Referrer-Policy strict-origin-when-cross-origin, Permissions-Policy, HSTS (prod only).
- File-upload size cap 10 MB; allowed mimes whitelist (`image/png|jpeg|webp|svg+xml|x-icon`, `application/pdf` for certifications).
- Public inquiries form sanitized via `isomorphic-dompurify`.
